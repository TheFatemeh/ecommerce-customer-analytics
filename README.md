# Olist E-commerce Analytics

Exploratory data analysis and interactive dashboard built on the [Olist Brazilian E-commerce Dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce).

## Setup

```bash
conda env create -f environment.yml
conda activate olist-dashboard
```

## Generate the data

The dataset is downloaded from Kaggle and the processed CSVs are generated by running the first notebook. Do this once before launching the dashboard.

```bash
jupyter notebook 01_data_cleaning_and_preparation.ipynb
```

Run all cells. This creates the `data/processed/` folder with the CSVs the dashboard depends on.

## Run the dashboard

```bash
streamlit run dashboard.py
```

Opens at `http://localhost:8501`.

## Project structure

```
├── 01_data_cleaning_and_preparation.ipynb   # Data loading, merging, saving processed CSVs
├── 02_exploratory_analysis_and_business_questions.ipynb  # EDA and business insights
├── dashboard.py                             # Streamlit dashboard (5 pages)
└── data/processed/                          # Cleaned CSVs used by the dashboard
```

## Key findings

- Late deliveries are 6.7% of orders but drive 36.6% of all 1-star reviews (5.5× over-represented)
- Repeat customers spend 2× more ($327 vs $161 avg) but represent only 3.1% of the customer base
- Black Friday 2017 permanently lifted monthly revenue from ~$400K to a ~$1M+ plateau
